FORWARD PASS DATAFLOW

ONE-HOT ENCODING

Each input position has a 3-way dial. The dial physically sets one of three binary lines to HIGH (brass rod pushed up) and the other two to LOW (rod stays down). This is one-hot encoding — and it eliminates multiplication in the forward pass entirely.

Why? Because multiplying by 0 or 1 is trivial: you either connect the weight or you don't. The lever only needs to sum the connected weights. No analog multiplication needed.

Key simplification: One-hot encoding means the forward pass is pure addition. Each neuron lever just sums the weights corresponding to the active inputs. The worm gear positions ARE the weights — nothing is multiplied, just selected and summed.
THE 42-WEIGHT GRID (7 × 6)

All 42 worm gears are arranged in a visible 7 × 6 grid on the front face. This is the machine's "brain" — every weight is visible, touchable, and its current position tells you what the machine has learned.

TRAINING DATA (PIN DRUM)

The pin drum encodes complete sentences. Each row of pins represents one training example: 9 input pins (the sentence) + 3 target pins (the correct answer). The drum holds ~8-12 training examples per revolution.

Example corpus — "Which animal does this?":

Sentences (Input)

Cat Eats Fish → Cat
Dog Chases Mouse → Dog
Bird Eats Seed → Bird
Cat Sees Mouse → Cat
Dog Eats Fish → Dog
Bird Sees Seed → Bird

What the Machine Learns

After training, the machine discovers: the first word (subject position) determines the answer. The verb and object are distractors.

Test it: set dials to "Cat Chases Seed" (never in training data). The machine should still predict "Cat" — it has generalized.

This is real learning. Not lookup. The machine found a pattern.

Big arrow on the drum: A brass plaque reads "HERE IS THE TRAINING DATA — We're feeding it the answers before asking the questions." This makes the supervised learning paradigm physically visible and demystifies the entire process.
