Design for robustness: No component is thinner than 1.5mm. Worm gear teeth are oversized for 3D printing. Lever pivots use steel pins in brass bushings, not PLA-on-PLA. Every sliding joint can be lubricated.
BUILD SEQUENCE

The genius review recommended building in order of risk. Each phase validates the hardest unknowns first, so you never build the whole machine only to find one component doesn't work.

Phase 1: One Weighted Lever

Prototype first, validate everything

Build one neuron: beam, fulcrum, 3 sliding blocks, mechanical stop (ReLU), one input rod. Verify: does the beam tip move proportionally to the weighted sum? Does the ReLU stop work? How much friction?

Phase 2: Worm Gear Grid (3 cells)

Test weight persistence

Build 3 worm gear cells. Verify: do they self-lock? Can error ratchets adjust them? How many clicks to traverse full range? Calibrate weight resolution.

Phase 3: Escapement + Motor

Establish the heartbeat

Build the clock escapement and connect to motor. Verify: consistent tick rate? Adjustable speed? Audible ticks? This sets the rhythm for everything else.

Phase 4: Forward Pass (1 hidden neuron)

First end-to-end signal path

Connect input selectors to one lever via brass rods through midplane. Verify: does a one-hot input produce correct weighted sum at beam tip? Signal routing works?

Phase 5: Full Forward Pass

All 6 neurons, all 42 weights

Build remaining 5 neurons. Connect all layers. Add answer prism. Verify: does inference work? Manual dials in, correct prediction out?

Phase 6: Backward Pass

Learning circuitry

Add whippletree (loss computation), error backpropagation through reversed levers, ratcheted worm gear updates. Verify: does the machine actually learn?

Phase 7: Pin Drum + Automation

Autonomous training

Build pin drum, dog clutch, epoch counter. Wire up autonomous training loop. Verify: does the machine train itself and converge?

Phase 8: Polish

Brachistochrone, dampers, labels

Add gradient ball track, eddy-current damper on whippletree, brass plaque ("HERE IS THE TRAINING DATA"), reset crank. Final tuning of tick rate, learning rate, and training data.

MATERIALS & AESTHETIC

The material palette is deliberate. Brass for signal and weight elements (warm, golden, catches light). Black PLA for structure (recedes visually). Clear acrylic for the midplane (invisible, reveals signal routing). Steel for precision shafts and pivots. Red for target pins on the drum (the answers, highlighted).

Material Used For Visual Role
Brass Signal rods, weight blocks, lever beams, input dials, drum frame, plaque Golden "nervous system" of the machine. Everything the eye should follow is brass.
Black PLA Frame, brackets, housing, worm bodies Disappears. Structure shouldn't compete with mechanism for attention.
Clear Acrylic Midplane, drum cylinder Invisible structure. Reveals what's inside. Brass rods cutting through crystal.
Steel Shafts, pivot pins, ball (brachistochrone), anchor (escapement) Precision elements. Dark, functional, strong.
Red PLA/Paint Target pins on drum, "wrong" indicator on whippletree Danger/answer color. Draws eye to training data targets.
The rule: If it computes, it's brass. If it supports, it's black. If it should be transparent, it's acrylic. If it needs precision, it's steel. If it's an answer, it's red.
THE PROMISE

When complete, this machine will sit on a table. A viewer will watch it train for 40 minutes — levers swinging, worms clicking, beams settling, a ball slowly rolling down a golden curve. Then they'll flip a brass lever (CLUNK), turn three dials, and watch the machine think: tick... tick... tick... click-click-CLUNK. The answer prism stops.

The machine predicted correctly. Something it was never explicitly told.

"AI is not magic. It's just mathematical probability."

And yet, watching 42 brass blocks slowly creep along their beams until the machine starts getting answers right... that IS magic. The mechanical kind.
